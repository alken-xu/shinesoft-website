## 1. プロジェクト概要

### 1.1 目的

株式会社シャインソフトの企業ホームページを全面リニューアルし、以下を実現する。

- 視覚的に高品質なデザインで企業ブランドを強化する
- バックエンドCMSによるコンテンツ（会社情報・ニュース）の自社更新体制を確立する
- 日本語・中国語・英語の3言語対応により、グローバルな採用・事業拡大を支援する
- レスポンシブデザインとSEO/GEO対策で集客・認知度を向上させる
- お問い合わせフォームのメール自動送信により、リード対応の効率化を図る

### 1.2 スコープ

| 項目 | 内容 |
|------|------|
| 対象 | shinesoft.co.jp 全ページ |
| 既存コンテンツ | 流用（後述のコンテンツ仕様に基づく） |
| 新規機能 | CMS、多言語、メール自動送信 |
| 対象外 | i-study.jp（別サイト、変更なし） |

## 3. システム要件

### 3.1 推奨技術スタック

| レイヤー | 推奨技術 | 理由 |
|---------|---------|------|
| フロントエンド | Next.js (App Router) | SSG/SSR両対応、SEO最強、i18n対応 |
| スタイリング | Tailwind CSS + Framer Motion | 高品質アニメーション・カスタムデザイン |
| CMS | Contentful / Sanity / Notion API | Markdown編集・多言語対応・API提供 |
| メール送信 | Resend / SendGrid / Nodemailer | 問い合わせフォームの自動送信 |
| ホスティング | Render | GitHub連携・自動デプロイ（`alken-xu/shinesoft-website`） |
| 言語ルーティング | Next.js 組み込み i18n | URLベースのロケール切替（デフォルト: ja） |

> **代替案：** CMSをWordPress（headless）とし、REST APIまたはGraphQLでフロントエンドと連携する構成も選択可能。

## 5. デザイン要件

### 5.1 デザインコンセプト

**「革新と信頼を体現する、洗練されたテックブランド」**

- シャインソフトの技術力・先進性・グローバル感を視覚的に表現
- ダーク基調 or ライト基調＋アクセントカラーの組み合わせで高級感を演出
- アニメーション・パーティクル・グラデーションを効果的に使用
- ユーザーが「ここに仕事を任せたい」「ここで働きたい」と感じるデザイン

### 5.2 デザイン仕様

| 項目 | 仕様 |
|------|------|
| カラーパレット | プライマリ：ディープネイビー (#0F172A) / アクセント：シャインゴールドまたはシアン (#38BDF8 or #F59E0B) |
| タイポグラフィ | 日本語：Noto Sans JP / 英語・中国語：Inter + Noto Sans SC |
| アニメーション | スクロールトリガーアニメーション（Framer Motion）、ヒーロービジュアルのグラデーション/パーティクル |
| 画像スタイル | 高解像度写真・WebP形式・遅延ロード |
| アイコン | Lucide Icons / Heroicons（SVG） |

### 5.3 ページ別デザイン要点

**トップページ (/):**
- フルスクリーンヒーロービジュアル（パーティクルアニメーション背景）
- キャッチコピー：「技術革新、自己変革を加速させる。」
- ミッション・ビジョン・サービス・ニュース・採用誘導セクション
- カウンターアニメーション（社員数・プロジェクト実績数など）
- ※「Innovation from Imagination」バッジおよびスクロールインジケーターは削除済み

**各サービスページ:**
- ページ遷移アニメーション
- サービス内容をアイコン＋テキストで視覚的に整理
- CTAボタン（お問い合わせへの誘導）

**採用情報:**
- 社員の声・雰囲気が伝わる写真・イラスト配置
- 職種別のカード型レイアウト

## 6. 多言語対応要件

### 6.1 対応言語

| 言語 | コード | 対象ユーザー |
|------|-------|------------|
| 日本語 | ja | 国内企業・求職者 |

| 中国語（簡体字） | zh | 中国語圏エンジニア採用・中国系企業 |

- 全ページの静的テキスト（ナビゲーション・ヘッダー・フッター・各ページ本文）
- CMSコンテンツ（ニュース・会社情報）は言語別エントリーで管理
- お問い合わせフォームのラベル・エラーメッセージ
- メタタイトル・メタディスクリプション（言語別に最適化）

### 6.3 翻訳ワークフロー

- 初回翻訳：プロ翻訳またはAI翻訳（DeepL）＋ネイティブチェック推奨
- CMS登録：日本語・英語・中国語を各フィールドに入力
- i18nファイル形式：JSON（next-intl標準）

## 8. SEO・GEO対策要件

### 8.1 技術的SEO

| # | 対策項目 | 詳細 |
|---|---------|------|
| S-01 | メタタイトル・ディスクリプション | 全ページ個別設定・キーワード最適化 |
| S-02 | hreflangタグ | 多言語ページ間の正規化 |
| S-03 | XMLサイトマップ自動生成 | /sitemap.xml を動的生成・Search Console登録 |
| S-04 | robots.txt | クローラー制御ファイル設置 |
| S-05 | 構造化データ (JSON-LD) | Organization / BreadcrumbList / NewsArticle / FAQPage |
| S-06 | canonical URLタグ | 重複コンテンツ対策 |
| S-07 | OGP / Twitter Card | SNSシェア最適化 |
| S-08 | ページ速度最適化 | 画像WebP化・遅延読込・コードスプリット |
| S-09 | HTTPS対応 | SSL証明書設置（必須） |
| S-10 | URL正規化 | スラッシュ有無・大文字小文字の統一 |

### 8.2 GEO（Generative Engine Optimization）対策

AIによる検索回答（Google SGE, ChatGPT, Perplexityなど）に自社情報が引用されやすくする施策。

| # | 対策項目 | 詳細 |
|---|---------|------|
| G-01 | E-E-A-T強化 | 代表者・専門家プロフィール、実績・認証資格の明示 |
| G-02 | 構造化FAQページ | よくある質問をFAQページで整備（FAQPage構造化データ付き） |
| G-03 | 権威ある外部引用 | CNCF・KCSP・KTP・ISMS等の認証情報を明記、外部ソースへのリンク |
| G-04 | コンテンツの網羅性 | 各サービスページに詳細な説明・事例・メリットを記載 |
| G-05 | About Usページ強化 | 会社の歴史・ミッション・ビジョン・実績を詳細に記述 |
| G-06 | ニュース・ブログの定期更新 | AIは最新情報を優先するため、定期的な情報発信が重要 |
| G-07 | NAP情報統一 | 会社名・住所・電話番号をすべてのページ・外部ディレクトリで統一 |
| G-08 | Wikipedia/業界サイト連携 | 外部権威サイトからのメンション・被リンク獲得 |

## 9. お問い合わせ・メール自動送信要件

### 9.1 フォーム仕様

| フィールド | 種別 | 必須 | バリデーション |
|-----------|------|------|--------------|
| お問い合わせ内容 | セレクトボックス | ○ | 必須選択 |
| 会社名 | テキスト | ○ | 最大100文字 |
| 氏名 | テキスト | ○ | 最大50文字 |
| メールアドレス | メール | ○ | メール形式チェック |
| 電話番号 | テキスト | ○ | 数字のみ・10〜11桁 |
| 備考 | テキストエリア | ✗ | 最大1000文字 |
| プライバシーポリシー同意 | チェックボックス | ○ | 必須チェック |

**お問い合わせ内容の選択肢：**
- SES
- ソフトウェア受託開発
- 労働者派遣
- リクルート
- その他

**多言語対応：** フォームラベル・エラーメッセージ・プレースホルダーは3言語対応

### 9.2 メール送信仕様

**自動返信メール（送信者宛）:**
```
件名: 【シャインソフト】お問い合わせを受け付けました
送信元: noreply@shinesoft.co.jp
内容: 受付完了のお知らせ、お問い合わせ内容の確認コピー、担当者からの返信目安
```

**社内通知メール（担当者宛）:**
```
件名: 【新規お問い合わせ】{お問い合わせ内容} - {会社名}
送信先: 担当メールアドレス（環境変数で設定）
Reply-To: 問い合わせユーザーが入力したメールアドレス（担当者が返信する際に直接ユーザーへ届くようにするため）
内容: 全フォーム入力内容、送信日時、IPアドレス（スパム対策用）
```

### 9.3 スパム対策

- reCAPTCHA v3（バックグラウンド検証、スコア0.5以上を有効とする）
- Honeypot フィールド（ボット対策）
- レートリミット（同一IPからの連続送信を制限）

### 12.6 クラウドサービス (/services/cloud)

```markdown
## クラウドサービス

### クラウドサービスの活用が重要

これまでITインフラの整備・構築にはオンプレミスによる運用が一般的でしたが、現在はクラウドサービスの活用が主流となっています。

### クラウド構築

インターネット上のサーバを利用し、業務システムなどをクラウドで運用できる状態にセットアップを行います。インフラをクラウド化することで、業務の効率化やコストダウンが実現されます。

### サーバ仮想化

サーバ仮想化とは、ソフトウェアによって複数のハードウェアを統合することで、1台のサーバを複数台のように作動させる技術です。ベースとなる1台を「物理サーバ」、仮想化されたものを「仮想サーバ」と呼びます。

導入により、サーバ台数を大幅に削減できコストカットにつながり、ハードウェアリソースを効率的に利用できるようになります。

### 自社技術資格

- KCSP（Kubernetes認定サービスプロバイダー）
- KTP（Kubernetesトレーニングパートナー）
- ISMS認証（ISO/IEC 27001:2013）
```
